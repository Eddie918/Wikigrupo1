Requerimientos

Requerimientos Funionales

  1. Gestión de cuentas de usuario:

    • Los usuarios pueden registrarse como arrendatarios o arrendadores.
    • Sistema de autenticación seguro para acceder a las cuentas de usuario.
    • Los arrendatarios y arrendadores tienen diferentes permisos y acceso a funcionalidades específicas.
    • Se requiere la validación de la dirección de correo electrónico durante el registro.

  2. Gestion de propiedades:

    • Los arrendatarios pueden crear, editar, eliminar y actualizar las propiedades disponibles para arrendamiento.
    • Los arrendadores pueden ver las propiedades disponibles, filtrarlas y solicitar el arrendamiento.

  3. Interacción entre Usuarios:

    • Los arrendatarios pueden revisar y responder a las solicitudes de arrendamiento recibidas.
    • Los arrendadores pueden hacer solicitudes de arrendamiento.

  4. Gestión de pagos:

    • Los arrendatarios pueden verificar los pagos recibidos por los arrendadores.
    • Los arrendadores pueden realizar pagos por el arrendamiento de las propiedades.

  5. Calificaciones y Comentarios:

    • Los arrendatarios pueden proporcionar calificaciones y comentarios sobre la experiencia de arrendamiento.
    • Los arrendadores pueden calificar tanto la propiedad arrendada como al arrendatario.

Requerimientos no funcionales

  1. Seguridad:

    • Sistema de autenticación y autorización robusto para proteger la información del usuario y garantizar la integridad de los datos..

  2. Usabilidad:

    • Interfaces de usuario intuitivas y funcionales para una experiencia de usuario fluida y satisfactoria.
    • Disponibilidad de ayuda y guías para los usuarios en caso de necesitar asistencia.

  3. Rendimiento:

    • Tiempo de respuesta rápido para las solicitudes de búsqueda de propiedades y procesamiento de transacciones de arrendamiento.
    • Capacidad para manejar múltiples usuarios concurrentes sin degradación del rendimiento.

  4. Escalabilidad:

    • Capacidad para manejar un crecimiento futuro en el número de usuarios y propiedades sin comprometer el rendimiento o la funcionalidad.

  5. Fiabilidad:

    • Minimización de errores y fallos del sistema mediante pruebas exhaustivas y monitoreo continuo.
    • Respaldo de datos para evitar pérdida de información crítica.

  6. Compatibilidad:

    • Compatibilidad con diferentes dispositivos y navegadores web para garantizar el acceso desde una variedad de plataformas.
    • Adaptabilidad a diferentes tamaños de pantalla para una experiencia consistente en dispositivos móviles y de escritorio.
